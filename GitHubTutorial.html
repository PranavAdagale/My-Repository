Git and GitHub Tutorial

Git is a version control system. Version control system is a tool that helps to track changes in a code.

Github is a website that allows developers to store and manage their code using Git.

How to use Git?

Git is a terminal that allows user to pull or push the code from remote repo(repos on github) to local machine(repos on user pc) or vicevrsa.

1. Download git bash

on the pc. After installing it we can check its version using git --version command.

2. Configuring git

It means that after after making on git files, from which name or email should the applied changes show.

There are types of git-config 1]Local- means we are making changes in specific file of repo from specific email or different email. 2]Global- If we want to use only one email not different then we can go with global...let's go with global one.

git config --global user.name "GitHub_account_uname

git config --global user.email "GitHub_account_email

Configured Successfully!

To get more info about configuration or configured or not.
git config --list

Commands can be types on git bash but it is better to use them on editors terminal.

3. clone(pulling) and Status

clone- git clone url
Here url is the link of the repo located on github account or we can also type git clone https://github.com/Username/RepositoryName

Status- git status
 Shows state of coding file. 4 kinds of status can be seened-
i.Untracked If we create new file inside folder of clonned github repo then we will see this status cause we created it on github but it is not added on github.
ii.Modified When we change the code but not added on github.
iii.Unmodified Unchanged(normal).
iv.Staged File is ready to be commited.

4. Add and Commit

add- git add file_name OR git add . where . means all files in current folder Adds new or changed files of users working directory to git staging area.
commit- git commit -m "some comment" It is the record of change.

Now change has been recorded but to update this change on github repo we use push command.

5. Push

git push origin main Uploads local repository content to remote repository.
